Useful Tags

  • <html> - Defines the root of an HTML document
  • <head> - Contains meta information about the document
  • <body> - Contains the content of the HTML document
  • <meta> - Provides metadata about the HTML document
  • <header> - Represents introductory content or a set of navigational links
  • <h1> to <h6> - Define HTML headings
  • <p> - Defines a paragraph
  • <ul> and <li> - Define an unordered list and list items
  • <a> - Defines a hyperlink
  • <img> - Embeds an image in the document

SEO Meta Tags

Important meta tags for search engine optimization and social media sharing:

  • <title> - The most important SEO tag. Appears in browser tabs and search engine results. Should be descriptive and include relevant keywords.
  • <meta name="description"> - Provides a summary of the page content. Shows up as the preview text under the title in search results.
  • <meta name="viewport"> - Essential for responsive design. Controls how the page is displayed on mobile devices. Typically set to "width=device-width, initial-scale=1.0".
  • <meta charset> - Specifies the character encoding for the document. UTF-8 is the standard and supports all characters.
  • <meta name="author"> - Identifies the author of the page. Useful for content attribution.
  • <meta name="keywords"> - Lists keywords related to the page content. Note: Most modern search engines ignore this tag, but it can still be included.
  • <meta name="robots"> - Controls how search engines index and follow links on the page.
  • Open Graph tags (<meta property="og:*">) - Used for social media sharing (Facebook, Twitter, LinkedIn). Controls how the page appears when shared, including title, description, and image.
  • <link rel="canonical"> - Helps prevent duplicate content issues by specifying the preferred URL for a page.
  • <link rel="icon"> - Specifies the favicon (favorite icon) for the page. The small icon that appears in browser tabs, bookmarks, and browser history. Common formats include .ico, .png, or .jpg files.

Positioning Demo

Static Positioning
Default positioning. Elements flow normally in the document. Use when you don't need special positioning.
Relative Positioning
Positioned relative to its normal position. Can use top/left/right/bottom to offset. Use for small adjustments or as a container for absolute children.
Absolute Positioning
Removed from normal flow, positioned relative to nearest positioned ancestor. Use for popups, dropdowns, or overlays that need precise placement.
Fixed Positioning
Positioned relative to viewport, stays in place when scrolling. Use for navigation bars, chat buttons, or elements that should always be visible.
